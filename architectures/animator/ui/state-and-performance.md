UI 상태 저장 범위와 성능 기준을 정의합니다.

## 상태 관리

- UI 영속 상태는 인스턴스 계층이 아니라 플러그인 로컬 설정에 저장합니다.
- 영속 상태:
  - 창 배치
  - 패널 열림/닫힘
  - 마지막 프로필
  - 사용자 기본 편집 옵션(스냅, 오토키 등)
- 세션 상태:
  - 현재 선택
  - 임시 툴 상태
  - 프레임별 미리보기 캐시

## 성능 기준

- Outliner/Timeline 행은 가상 스크롤을 사용합니다.
- 변경 구간만 부분 렌더링합니다.
- 커브/키프레임 대량 편집 시 배치 업데이트를 사용합니다.
- UI 이벤트와 런타임 적용 루프를 분리해 프레임 드랍을 줄입니다.
- `GraphEditor`는 줌 레벨 기반 키/핸들 표시 밀도를 조절합니다.
- `GraphEditor`는 선택되지 않은 채널을 저비용 샘플링으로 렌더링합니다.

## 의사결정 규칙

- 기본값은 단일 `EditorShell` 유지입니다.
- 완전 분리 에디터는 데이터 모델이 독립될 때만 고려합니다.
- 어댑터/UI 확장은 훅/프로필로 우선 해결합니다.
- 나머지 코어 원칙은 [UI](./index.md)의 `코어 원칙`을 단일 원본으로 참조합니다.
