런타임 문서의 공통 범위와 제공 정책을 정의합니다.

공통 레이어 기준과 충돌 우선순위는 [Architecture Main](../main.md)을 따릅니다.

## 범위

- 런타임 데이터 구조
- 런타임 평가/재생기
- 레이어 합성 규칙

이 폴더는 편집기 자체가 아니라, 편집 결과를 프로젝트 런타임에서 어떻게 소비할지에 대한
기본 구조와 책임 경계를 다룹니다.

## 제공 정책

- `architectures/runtime/**`는 플러그인이 자동으로 삽입하거나 강제 제공하는 내장 런타임 구현만을 뜻하지 않습니다.
- 이 문서들은 `DefaultRobloxAdapter`를 기준으로 가장 자연스럽게 맞물리는 기본 런타임 템플릿과 권장 계약을 정의합니다.
- 프로젝트는 이 구조를 그대로 구현할 수도 있고, 어댑터 계약을 유지한 채 자체 런타임으로 치환할 수도 있습니다.
- 즉, 플러그인은 편집기와 데이터 구조를 제공하고, `runtime` 문서는 "기본 구현 가이드" 성격을 함께 가집니다.

## 책임 경계

- [AnimationData](./animation-data/index.md)는 정적 런타임 데이터 구조를 정의합니다.
- [AnimationTrack](./animation-track/index.md)는 데이터 평가와 재생 상태를 정의합니다.
- [AnimationLayer](./animation-layer/index.md)는 레이어 합성 규칙을 정의합니다.

하위 문서는 이 공통 정책을 반복하지 않고, 각자 구조와 책임만 설명합니다.
